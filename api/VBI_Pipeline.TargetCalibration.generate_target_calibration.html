generate_target_calibration¶

VBI_Pipeline.TargetCalibration.generate_target_calibration(intsects, pos_list, refidx, dmax, header, output='Target_Calibration.fits', debug=False)[source]¶

Find overlapping intersetions and use them to compute sub-pixel offsets between pairs of hdus.

The resulting offsets are then saved in the metadata (i.e., header) of a data-less Data.FitsData object.

As of right now this algorithm is a little dumb; it only finds the offsets relative to a single, reference hdu. In the future it should be possible to use the information from all hdus that overlap.

Parameters:

intsects : tuple

Each element of this list corresponds to a single input hdu and contains a numpy array of shape (2, N) containing the (x,y) WCS coordinates for the N intersections found in that hdu. Probably the output of collect_intersections()

pos_list : list

A list containing the camera positions (VBI__005) of each corresponding hdu. Probably the output of collect_intersections()

refidx : int

Index of the reference hdu to use. Index corresponds to the location in both intsects and pos_list

dmax : float

Any intersections with pixel locations less than dmax apart are considered to be the same intersection.

header : astropy.io.fits.header.Header

Header of the line grid data set used to compute this Target Calibration

output : str

Location to save the resulting Target Calibration. It’s a little strange that this is in this function instead of main()

debug : bool

If true then generate a plot called “overlap.pdf” that shows all the intersections with those used to compute the shifts circled in red.
Returns:

lst

Length N list where N is the number of hdus and the elements are numpy arrays with [x_shift, y_shift]
