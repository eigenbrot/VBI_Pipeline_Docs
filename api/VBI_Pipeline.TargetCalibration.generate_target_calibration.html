VBI_Pipeline.TargetCalibration.generate_target_calibration(intsects, pos_list, refidx, dmax, header, output='Target_Calibration.fits', bimode_choice=False, debug=False)[source]Â¶

Find overlapping intersections and use them to compute sub-pixel offsets between pairs of hdus.

The resulting offsets are then saved separate HDUs of a Data.FitsData object.

As of right now this algorithm is a little dumb; it only finds the offsets relative to a single, reference hdu. In the future it should be possible to use the information from all hdus that overlap.

Parameters
intsectslist

Each element of this list corresponds to a single input hdu and contains a numpy array of shape (2, N) containing the (x,y) WCS coordinates for the N intersections found in that hdu. Probably the output of collect_intersections()

pos_listlist

A list containing the camera positions (VBI__005) of each corresponding hdu. Probably the output of collect_intersections()

refidxint

Index of the reference hdu to use. Index corresponds to the location in both intsects and pos_list

dmaxfloat

Any intersections with pixel locations less than dmax apart are considered to be the same intersection.

Header of the line grid data set used to compute this Target Calibration

List containing the automatic choice for potential bimodality in the (x, y) dimensions. Choice can be either 1 or 2; 1 always corresponds to the mode closest to the global median.
